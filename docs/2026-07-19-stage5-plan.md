## 1. Why Stage 5

Stage 4 made monitor **single-durable**. Remaining work:

1. **Ops hygiene** — archived dead-letters, zeta observation, CUA timeout class.
2. **Code surface honesty** — offline `generate_reply` stays but must not look “live”; thin-monitor UI must not imply dual-path.
3. **Control console align** — Streamlit still teaches pre-Stage-4 mental model in places.
4. **Ops gates clarity** — multi-target works; allowlist/opt-in/rollback must be visible and correct in UI.

Not a second transport migration.

### 5F. Streamlit control console align — **in scope (user #4)**

**Audit snapshot (2026-07-19 `wechat-control-ui/app.py`):**

| Area | Issue vs Stage 4 |
|------|------------------|
| Pages | 总览 / 监听目标 / 触发词 / 知识库 / Agent 任务 — **no first-class Reliable Pipeline page** |

3. **P2 — Overview cleanup (should)**  
   - Prefer durable telemetry over legacy aggregator reasons where available.  
   - Caption: Stage 4 single-durable; link to this plan.

4. **P3 — Agent page trim (nice)**  
   - Keep auto pool; demote “旧同步 worker” further.  
   - Clarify test jobs are **not** WeChat live dual-path.

- Operator cannot enable a “thin monitor dual-path” thinking it is production.
